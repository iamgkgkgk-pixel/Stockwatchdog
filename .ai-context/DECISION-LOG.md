## DEC-001: 使用方案B统一基准（本地 calcPercentile 替代 API pePercentile）

- **日期**: 2026-03-22
- **背景**: 
  - 实时信号用蛋卷基金 API 返回的 `pePercentile`（基于全市场 10 年+数据），恒生科技 PE=25.8 → 分位 55.2%
  - 历史回算用 `calcPercentile(pe, historicalPE)`（仅 JSON 中 63 个月数据），同一 PE → 分位 77.8%
  - 同一个 PE 值，两套基准差 22.6 个百分点 → 综合评分差 9 分 → 信号级别可能跳两级
- **决策**: 统一使用 `calcPercentile(当前PE, JSON全量PE)` 计算分位，放弃 API pePercentile
- **影响**: 所有分位计算走同一条路径（calcPercentile），历史走势与实时信号 100% 一致
- **⚠️ 不可逆转**: 如果改回 API 分位，历史走势会再次出现断崖式不一致

## DEC-003: 历史走势图 marketTemp 统一为 50

- **日期**: 2026-03-22
- **背景**:
  - 历史月份无法获取真实的 CNN F&G 或 A 股广度数据
  - 如果 marketTemp 设为 null，sentiment 维度被跳过（权重 20-25%），历史月份只用 75-80 权重计算，当前月用 100 权重 → 基准不同
  - 如果部分月份有 marketTemp、部分没有 → 走势图曲线断裂
- **决策**: 所有历史月份 marketTemp 统一为 50（中性），走势图只反映估值+安全边际的变化趋势
- **影响**: 走势图中 sentiment 维度对所有月份贡献一致（50 分），消除了基准差异
- **⚠️ 不可逆转**: 如果给不同月份设不同的 marketTemp，走势图曲线会因基准不同而产生虚假的趋势变化

## DEC-004: 安全边际从线性公式改为 E/P 倒数公式

- **日期**: 2026-03-22
- **背景**: 原公式 `130 - PE * 2` 对高 PE 行业（科创板 PE 50-80）不公平，PE=80 → 安全分 -30（被 clamp 到 0）
- **决策**: 改为 `E/P = (1/PE) × 100`，然后线性映射到 0-100
  - PE=20 → E/P=5% → ~75 分
  - PE=40 → E/P=2.5% → ~55 分
  - PE=80 → E/P=1.25% → ~30 分
- **影响**: 成长股（科创、创业板、半导体等）的安全边际评分更合理，不再一刀切扣分

## DEC-006: localStorage 缓存优先级 — JSON 预设 > 缓存

- **日期**: 2026-03-22
- **背景**: 如果 localStorage 中有旧的脏数据（如错误的 PE 值），会覆盖 JSON 预设的正确数据
- **决策**: `normalizeData` 中，JSON 预设数据优先级高于 localStorage 缓存
- **影响**: 即使 localStorage 有数据，也以 JSON 文件为准，localStorage 只在 API 和 JSON 都不可用时才作为最终回退

## DEC-010: 综合分历史分位图表不混入实时综合分

- **日期**: 2026-04-16
- **背景**: 
  - 第一次修复分位图表不一致时，将实时综合分（含真实 marketTemp）追加到图表曲线末尾
  - 结果医药 ETF 从绿色突变为橙红色（实时分 43.1 vs 图表统一基准分 65.5）
  - 用户反馈："请不要为了4者完全一致，而引入错误的引导！"
- **决策**: 
  - 图表保持纯净（全部基于 marketTemp=50 统一基准）
  - 摘要和标记点使用图表同基准的 `lastSignal.score`，不使用 `currentTotal`（实时综合分）
- **教训**: 数据一致性 > 表面一致性。不同基准的数据混合比标注差异更危险
- **⚠️ 不可逆转**: 混入实时综合分会导致最后一个点与历史数据不可比，用户无法判断趋势

## DEC-011: 利差安全边际引入利率环境因子

- **日期**: 2026-03-22
- **背景**: 低利率环境下（国债收益率 < 2%），股债利差天然偏大，安全边际分虚高
- **决策**: 引入利率调节因子 `rateAdj = max(0.7, min(1.0, (bondYield - 1.0) * 0.3 + 0.7))`
  - 国债 < 2% → 系数 0.7（压缩 30%）
  - 国债 > 3% → 系数 1.0（不压缩）
- **影响**: 价值型 ETF（红利低波、沪深300、自由现金流）的安全边际更客观

## DEC-012: 混合估值分 — 偏离度 70% + 分位 30%

- **日期**: 2026-03-22
- **背景**: 纯 PE 分位只回答"排第几"，但如果数据样本量小（63个月），分位不够稳定
- **决策**: `calcHybridValuationScore = deviationScore × 0.7 + percentileScore × 0.3`
  - 偏离度（deviation）: 衡量"离均值多远"，需要 peMean 和 peStd
  - 分位（percentile）: 衡量"排第几"，基于全量 PE 历史
- **理论依据**: 芒格"多把尺子"理论 — 两把尺子比一把尺子更可靠
- **影响**: 需要 JSON 数据中提供 `valuationAnchor: { peMean, peStd }`
